## El Dolor: "Se ve roto en mi celular"

Más del 60% del tráfico web es móvil. Si tu aplicación tiene botones inalcanzables, textos minúsculos o scroll horizontal indeseado en celulares, estás perdiendo a la mayoría de tus usuarios.

## La Solución: Mobile-First Design System

Utilizamos **Tailwind CSS** para construir interfaces fluidas que se adaptan matemáticamente al viewport. No hacemos "versiones móviles", hacemos un único diseño que fluye.

### Características Clave

*   **Breakpoints Inteligentes:** Layouts que cambian drásticamente (de fila a columna) según el espacio disponible.
*   **Touch Targets:** Botones y áreas de interacción diseñadas para dedos (mínimo **44px**), no solo para cursores de mouse.
*   **Modo Oscuro Automático:** Detectamos la preferencia del sistema operativo del usuario para ofrecer una experiencia visual descansada (ideal para dashboards de uso intensivo).

```css
/* Ejemplo conceptual de adaptación */
.grid-container {
  grid-template-columns: repeat(1, 1fr); /* Móvil: 1 columna */
}

@media (min-width: 768px) {
  .grid-container {
    grid-template-columns: repeat(3, 1fr); /* Desktop: 3 columnas */
  }
}
```

### Resultado

Una aplicación que se siente nativa en cualquier dispositivo, aumentando la retención de usuarios y la conversión.
