## El Dolor: El Mercado Limitado

Quieres vender en el extranjero o recibir turistas, pero tu plataforma solo habla español. Usar el traductor automático de Google suele romper el diseño y traducir términos técnicos erróneamente, dando una imagen poco profesional.

## La Solución: Routing Internacionalizado

Implementamos **Next-Intl** o librerías robustas de i18n integradas en el enrutamiento.

### Características

1.  **URL Semánticas:** `/es/contacto` vs `/en/contact`. Esto es crucial para que Google indexe cada versión por separado y muestre el idioma correcto según el país del usuario.
2.  **Detección Automática:** Redirigimos al usuario a su idioma preferido basándonos en la configuración de su navegador (`Accept-Language` header).
3.  **Gestión de Contenido:** Archivos JSON de traducción separados del código, permitiendo que traductores profesionales trabajen sin tocar la programación.

```json
// Ejemplo de estructura de locales (es.json)
{
  "HomePage": {
    "title": "Bienvenido al Futuro",
    "cta": "Empieza Ahora"
  }
}
```

### Resultado

Una plataforma global lista para escalar a cualquier mercado sin deuda técnica.
